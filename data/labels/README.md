# 目視ラベルCSVの置き場所

`data/labels/<target>/bin_labels.csv` に、Stage4（スコア統合・閾値決定）で使う
5秒ビン単位の目視ラベルを配置する。**動画の選定・完全目視・train/val分割は
ユーザー側で行う**。パイプライン側（`stage4_fit_score_model.py`）はこのCSVを
読むだけで、分割ロジック（GroupKFold等）は実装しない。

## スキーマ

| 列 | 説明 |
|---|---|
| `video` | 動画ID（Stage2/3出力の`video`列と一致させる） |
| `bin_start_sec` | ビン開始秒（5秒固定ビンの境界） |
| `bin_end_sec` | ビン終了秒 |
| `label` | 0または1（このビンに訪花対象が写っているか） |
| `split` | `threshold_decision`（閾値決定用）または`validation`（検証用）。
  仕様書§5の通り、分割は動画・撮影日・地点・連続時間区間単位で行い、ビン単位では行わない
  （＝同一動画の全ビンは同じsplit値になる想定） |

## 動作確認用ダミーデータ

本番ラベルがまだ無い状態でもコードパスを疎通確認できるよう、
`src/stage4_dummy_data_from_bombus.py`が既存のbombusバウト比較データ（バウトをビンと
読み替えたもの）から同スキーマのダミーCSVを生成できる。本番の分割方針の代用ではなく、
あくまでコードパスの疎通確認用。
